  入口见 [论文一致最终部署主线.md](论文一致最终部署主线.md)

## 2. 整个复现流程摘要

当前仓库的完整复现流程可以压缩为 `Stage A -> K` 四段：

| 阶段 | 主要回答的问题 | 当前主线沉淀 |
| --- | --- | --- |
| `Stage A-D` | baseline、最小映射和早期恢复链路能否闭环 | 建立明文基线与 block-level 早期恢复底座 |
| `Stage E-G` | 复杂 attention、KeyMat 与层内结构表达能否恢复 | 把“能跑通”推进到“开始保留论文所需结构扰动” |
| `Stage H-I` | 哪些混淆表达在工程上仍可部署，且仍处于标准推理框架边界内 | 定义并验证“可部署混淆表达” |
| `Stage J-K` | 如何把可部署表达收口为标准 HF 工件与 release surface | 形成可交付工件、profile、契约与入口 |

## 5. 部署线与论文差异对照表

| 维度 | 原始论文目标 | Qwen 当前主线 | Llama 当前主线 |
| --- | --- | --- | --- |
| 最终语义根 | 一个同时强调 deployability、accuracy 与 privacy 的统一论文叙事 | `paper_consistent` | `paper_consistent` |
| 当前唯一 release 面 | 论文没有 `Stage K` 命名，但要求最终可部署工件 | `artifacts/stage_k_release` | `artifacts/stage_k_llama_release` |
| correctness 证据 | 最终部署物应有完整可用性证明 | `outputs/stage_k_release/correctness/default.json`、`outputs/stage_k_release/correctness/reference.json` 与 `outputs/stage_k_release/correctness_summary.json` 已给出唯一 release 面结果 | `outputs/stage_k_llama_release/correctness/default.json`、`outputs/stage_k_llama_release/correctness/reference.json` 与 `outputs/stage_k_llama_release/correctness_summary.json` 是当前 release 面证据入口 |
| 安全 / 攻击证据 | 论文主叙事要求同时讨论 `VMA / IMA / ISA` 等攻击面 | 唯一 release 面上的 `VMA / IMA / ISA` 已补齐；其中 `VMA / ISA` 为低风险，`paper_like IMA` 也已回到低风险 | release 语义已与 Qwen 对齐，但尚未形成与 Qwen 同强度、同口径的攻击评测闭环 |
| 当前最强结论 | 应能表述为“论文口径下的最终部署方案” | 已收口为论文一致最终部署线，且 paper-like `IMA` 与论文结论不再冲突；Qwen 也是当前仓库中最完整、最接近论文闭环、已基本完善的主线，但公共语料规模与混淆参数仍未完全同态到论文默认设置 | 已切到 Llama-3.2-3B-Instruct 论文一致部署线，工程语义与 Qwen 的 Stage J/K release 组织方式对齐；安全闭环仍弱于 Qwen |

## 6. 不再混用的历史语义

以下名词现在只保留历史证据价值，不再承担活跃主线职责：

- Qwen 的 `legacy`
- Qwen 的 `buffered redesign`
- Qwen 的 `standard-visible bridge`
- 已迁入 `docs/history/` 的旧阶段总报告、旧安全索引和旧使用说明

换句话说，当前不应再把 `bridge` 当 release correctness 语义，也不应把旧安全子域文档当作主线入口。
